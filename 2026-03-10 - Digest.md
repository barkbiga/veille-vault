## 🔥 Article majeur

### Documentation Fraud: a Verification Architecture Failure
**Pertinence : 4/5 — 🔄 CONTRARIAN**

1 document sur 16 traités dans les institutions financières montre des signes de fraude. Le signal fort : le problème n'est **pas** la qualité des outils de détection, mais **l'architecture des workflows de vérification**. Les équipes sécurité cherchent à améliorer le point de contrôle terminal quand la faille est structurelle, en amont.

**Implication pour les architectes API :** les API d'onboarding KYC exposent souvent des flux séquentiels où une validation en étape N est implicitement considérée comme acquise en étape N+2. C'est une violation du principe Zero Trust appliqué aux processus. La vérification doit être continue, pas point-in-time.

**Actions :**
- Auditer les trust assumptions dans les workflows KYC/API onboarding
- Modéliser les flux de vérification avec un regard Zero Trust (never trust, always verify à chaque step)
- Challenger les vendors sur la capacité d'orchestration continue vs validation one-shot

## ⭐ Articles significatifs

### Pentagon's Anthropic Ban — Wake-Up Call for CIOs (3/5)
Signal de gouvernance IA : la dépendance mono-vendor sur les modèles IA devient un risque de continuité classifié "supply-chain risk" par le gouvernement US. Pour les architectes déployant des agents IA sur des APIs bancaires/santé : anticiper une stratégie multi-model et inclure le risque "AI vendor shutdown" dans les BIA (Business Impact Analysis). Connexion directe avec la Non-Human Identity : les agents IA sont des identités non-humaines dont le cycle de vie (y compris la disparition) doit être géré.

### Going Passwordless in Healthcare (3.5/5)
Les CISOs santé identifient les shared workstations et la résistance clinicienne comme obstacles majeurs au passwordless. Angle pertinent : FIDO2/passkeys en mode "tap-and-go" pour environnements partagés. À aligner avec la modernisation HIPAA (finalisation ~mai 2026) qui rendra la MFA obligatoire. Signal fort pour les architectes IAM secteur santé.

### Modernizing HIPAA: Are You Ready? (3.5/5)
Première révision majeure de la HIPAA Security Rule depuis des décennies, potentiellement finalisée en mai 2026. Les nouvelles exigences (MFA, chiffrement, audit) créent un momentum réglementaire pour porter des projets Zero Trust et API security. Opportunité de positionnement pour les architectes travaillant sur les API de données de santé (SMART on FHIR).
