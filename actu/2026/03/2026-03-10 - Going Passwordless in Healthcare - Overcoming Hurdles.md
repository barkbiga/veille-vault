# Going Passwordless in Healthcare: Overcoming Hurdles

| | |
|---|---|
| **Pertinence** | 3.5/5 |
| **Catégorie** | IAM / Santé |
| **Source** | [BankInfoSecurity](https://www.bankinfosecurity.com/interviews/going-passwordless-in-healthcare-overcoming-hurdles-i-5528) |

## Résumé
Les CISOs santé peinent à déployer le passwordless face à la résistance des cliniciens, qui craignent une dégradation de l'accès aux systèmes de soins critiques. L'interview identifie les freins organisationnels et techniques : flux de travail interrompus, partage de postes (shared workstations), et intégration avec les EHR. Pertinent pour les architectes IAM/CIAM ciblant le secteur santé, notamment dans un contexte d'API de santé (FHIR, SMART on FHIR) où l'authentification forte sans friction est un pré-requis de conformité post-HIPAA modernisation. Lien avec le sujet passwordless + API access control.

## Actions recommandées
- [ ] Mapper les cas d'usage cliniques (shared workstations, urgences) avant de déployer du passwordless
- [ ] Évaluer FIDO2/passkeys en mode "tap-and-go" adapté aux environnements shared-device
- [ ] Aligner la roadmap passwordless avec les exigences HIPAA Security Rule update (finalisaton ~mai 2026)
