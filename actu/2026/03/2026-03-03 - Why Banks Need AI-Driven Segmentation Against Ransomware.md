# Why Banks Need AI-Driven Segmentation Against Ransomware

| | |
|---|---|
| **Pertinence** | 2.5/5 |
| **Catégorie** | Actu / Banking / Network Security |
| **Source** | [BankInfoSecurity](https://www.bankinfosecurity.com/interviews/banks-need-ai-driven-segmentation-against-ransomware-i-5524) |

## Résumé
⚠️ Source vendor (Akamai). L'interview soutient que le ransomware IA compresse les cycles d'attaque de semaines à minutes, rendant la microsegmentation manuelle insuffisante. La segmentation pilotée par IA permettrait de contenir les intrus, limiter les mouvements latéraux et protéger les données sensibles avant l'extorsion. Argument pertinent pour les banques mais fortement coloré par le positionnement produit Akamai.

## Actions recommandées
- [ ] Évaluer objectivement (sans biais vendor) les capacités de microsegmentation IA dans les environnements API banking pour limiter la blast radius en cas de compromission de service account
