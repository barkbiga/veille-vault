# Shorter Certificates, Tougher Compliance: Are You Prepared?

## Résumé
Le mouvement vers des certificats TLS à durée de vie plus courte (CA/Browser Forum, Apple poussant vers 47 jours) impacte directement les architectures API bancaires : mTLS, certificats clients pour FAPI, certificats des API gateways. La rotation automatisée des certificats devient critique pour les Non-Human Identities (NHI) comme les service accounts et les agents IA. Contenu webinar non détaillé, mais la thématique est hautement actionnable pour les secteurs régulés.

## Actions recommandées
- [ ] Inventorier tous les certificats utilisés dans les flux API (mTLS, FAPI, API gateway) et leurs durées de vie actuelles
- [ ] Évaluer la maturité de l'automatisation PKI (ACME, cert-manager) pour supporter des cycles de renouvellement < 90 jours
- [ ] Inclure la gestion des certificats dans la stratégie NHI (rotation des credentials non-humains)
