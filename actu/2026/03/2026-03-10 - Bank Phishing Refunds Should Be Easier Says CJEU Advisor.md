# Bank Phishing Refunds Should Be Easier, Says CJEU Advisor

| | |
|---|---|
| **Pertinence** | 2.5/5 |
| **Catégorie** | Régulation / Fraude bancaire |
| **Source** | [BankInfoSecurity](https://www.bankinfosecurity.com/bank-phishing-refunds-should-be-easier-says-cjeu-advisor-a-30949) |

## Résumé
Un conseiller de la CJUE préconise une politique "rembourser d'abord, poursuivre ensuite" pour les victimes de phishing bancaire ayant subi des transactions non autorisées. L'affaire concerne une ressortissante polonaise piégée sur une plateforme d'enchères en ligne. Pertinence limitée pour les architectes API security, mais signal réglementaire intéressant sur la responsabilité des banques dans la chaîne d'authentification forte (SCA/PSD2). Connexion indirecte avec l'efficacité réelle des mécanismes d'authentification forte et leur opposabilité légale.

## Actions recommandées
- [ ] Surveiller l'évolution jurisprudentielle CJUE sur la responsabilité bancaire post-SCA
- [ ] Évaluer si les logs d'authentification forte (FAPI/OIDC) sont suffisamment probants en cas de litige
