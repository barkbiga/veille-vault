## 🔄 Contrarian du jour

### Data Minimization Is Still an Underrated Security Control
**Pertinence : 3.5/5** | [Lien](https://www.bankinfosecurity.com/blogs/data-minimization-still-underrated-security-control-p-4049)

🔄 CONTRARIAN — Remet en question l'axiome implicite selon lequel tokenisation vault-based et chiffrement sont les réponses primaires aux risques data. L'article argumente que la minimisation des données collectées est un contrôle structurellement supérieur : elle réduit l'impact des brèches, affaiblit le levier ransomware et sécurise les pipelines IA. **Application directe** : les scopes OAuth trop larges dans l'open banking constituent une forme de sur-collecte systémique. Réduire les claims exposés dans les tokens est à la fois une mesure de data minimization ET de réduction de la surface d'attaque NHI.

**Actions :**
- [ ] Auditer les scopes OAuth/OIDC exposés par les APIs open banking : identifier les over-permissioned tokens
- [ ] Intégrer la data minimization comme critère de design dans les API contracts (OpenAPI)
- [ ] Évaluer l'impact sur les pipelines IA utilisant des données clients

## 🏥 Gouvernance > Outillage en Santé

### Digital Risk Is Now a Clinical Challenge
**Pertinence : 3.5/5** | [Lien](https://www.bankinfosecurity.com/blogs/digital-risk-now-clinical-challenge-p-4051)

Les brèches en santé résultent de gouvernance faible, d'accès excessifs et de vendors non supervisés — pas d'une défaillance technique isolée. Argument transposable aux architectures API santé (FHIR, HL7) où les identités non-humaines (agents IA, intégrations tiers) prolifèrent sans inventaire ni rotation de credentials.

**Actions :**
- [ ] Auditer les accès third-party sur les APIs FHIR et vérifier le principe de moindre privilège
- [ ] Inventorier et gouverner les NHI (service accounts, API keys) dans les environnements santé
